# WAF plan (post-MVP)

Статус: отложено до этапа после MVP.

## Цель

Сделать управляемую WAF-защиту через админ-панель и системные настройки без пересборки приложения.

## Текущая база

- Есть rate-limit middleware на backend.
- Счетчики хранятся в Redis.
- Конфигурация системных настроек хранится в `system_setting` (`value` как JSONB).

## План развития

1. Ввести единый ключ настройки `security.waf` (JSON) как источник политики.
2. Добавить серверную валидацию JSON-схемы WAF (режимы, лимиты, списки IP и т.д.).
3. Перевести лимиты с hardcoded значений на чтение из `security.waf`.
4. Добавить уровни счетчиков:
   - per IP + route,
   - per user,
   - global per IP (общий).
5. Добавить профили (`default`, `high_load`, `incident`) и активный профиль.
6. Добавить расписание для переключения профилей (scheduler/cron + audit).
7. Добавить monitor mode/dry-run и audit-события по срабатываниям WAF.
8. Поддержать кэш конфигурации WAF в Redis с коротким TTL.

## Хранение

- Источник истины: таблица `system_setting`, ключ `security.waf`.
- Формат: JSONB в поле `value`.
- Изменения через `/admin/settings/security.waf` с audit-логом.
